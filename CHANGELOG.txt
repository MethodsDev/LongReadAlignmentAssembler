v0.12.7, November 18, 2025
- merge_LRAA script allows for alternative backends but defaults to in-memory
- bugfix: MultiPath assumed every read name could be resolved to a compact ID via READ_NAME_STORE. On Terraâ€™s merge job that store failed to initialize, so the fake provenance names for input transcripts never mapped to IDs, _read_count stayed zero, and MultiPathCounter.add tripped assert multipath_obj.get_read_count() > 0. We hardened MultiPath to fall back to deterministic hash-based IDs when the store is unavailable and to seed the count from the provided read_names, so the counter always sees a positive weight even in store-less merge mode.
- added wdl progress monitoring to cell cluster-guided lraa
- allow for gene_id and transcript_id in ref_gtf to be separated by other metadata
- sparse matrix builder - updating to avoid the segfault pandas bug, use python engine

v0.10.0 November 2, 2025
- Defaults now tuned for non-HiFi long reads (e.g., ONT). Replaced --LowFi flag with --HiFi to enable more stringent PacBio HiFi parameters.
- HiFi mode sets: min_per_id=98, infer_TSS/PolyA enabled, max_exon_spur_length=13, no splice-boundary aggregation, and fractures splice graph at input transcript bounds.
- runs --CPU parallel contig/strand combinations. If --no_parallelize_contigs, then allows up to --CPU per cluster of reads in per contig/strand evals.
- can resume from earlier partially completed runs.  Disable with  --no_resume_parallel
- uses SQLITE (or lmdb) where needed to reduce RAM usage.  See --store_backend
- new --bam_list option intended for quant-only runs with single cell cluster data, each bam corresponding to the cell cluster partitioned bam. Used in single cell pipeline (WDL).
